### Coaching: The Problem + позиционирование
- Боль зафиксирована со стороны **получателя** (нет прозрачности «началась доставка / когда приедет») — основание: личный опыт основателя как покупателя (цветы, пицца). Известно (known), не assumption.
- **Beachhead = пицца/еда** (не цветы): еда остывает → время критично.
- **Денежный мотив магазина (known рынок):** Wolt/Glovo в Сербии берут ~30% → кафе массово держат свою доставку, но потеряли клиентский трекинг/ETA. Позиционирование: «опыт уровня Wolt/Glovo для своих курьеров без 30% комиссии».
- **Разрыв «кто платит» закрыт частично:** мотив магазина = качество (горячая еда → повторные заказы) + экономия на комиссии. Остаётся подтвердить остроту на реальных магазинах.
- **ОТКРЫТО (определяет Solution/Scope):** оборачиваемость транспорта — ядро MVP или Vision? Уведомление влияет на оборот лишь на последней миле; системный выигрыш = маршрутизация/диспетчеризация (отдельный больший продукт).
- Черновик: заполнены Problem, What Makes This Different, Who This Serves.

### Coaching: Solution / Scope / Metrics
- **MVP = прозрачность + качество + быстрая передача.** Оборачиваемость/диспетчеризация → Vision (подтверждено заказчиком: «оборачиваемость не так важна сейчас»).
- Главные метрики успеха: **удержание магазина** + **готовность платить**.
- НОВАЯ фича/метрика по запросу заказчика: **оценка доставки получателем 1–5** на трекинг-странице после доставки. Взята в MVP. Требует минимальной отметки «Доставлено» магазином (статус, без отдельного push-уведомления) для триггера оценки.
