# Javi — EXPERIENCE

Визуальные токены — в `DESIGN.md` (ссылки вида `{path.to.token}`). При конфликте побеждают спайны, не макет.

## Foundation
- **Форм-фактор:** мобайл-first веб. Стек — Django server-rendered + лёгкий JS (HTMX/Alpine для частичных обновлений: старт доставки, отправка оценки, обновление статуса без полной перезагрузки).
- **Две поверхности:** (1) **Кабинет** магазина/курьера — авторизован, светлая утилита, использование «на бегу» (улица, кухня, одной рукой); (2) **Публичная страница статуса** получателя — без приложения и логина, по unguessable-токену, брендовая.
- Язык по умолчанию — сербский (латиница); en/ru — позже.

## Information Architecture

**Кабинет (авторизован):**
- **Вход** — email + пароль.
- **Главный · Доставки** — список «сегодня», сгруппирован: **U dostavi** → **Spremno** (готово к старту) → **Završeno**. Постоянная **＋ Nova dostava** внизу.
- **Nova dostava** — форма: имя, телефон, полный адрес получателя (+ описание опц.) → геокод → запись попадает в «Spremno».
- **Деталь доставки** — статус, ETA, чипы доставлено/прочитано, оценка; действие **Dostava je počela**; при сбое — правка номера + **Pošalji ponovo**.
- **Profil / Prodavnica** — адрес магазина (origin), задаётся один раз.

**Публичная (без логина):**
- **Status `/t/<token>`** — бренд-шапка (магазин + приветствие + статус) → степпер (Primljeno · U dostavi · Isporučeno) → крупный **ETA «do HH:MM»** → после доставки блок **оценки 1–5** → состояние «Hvala!».

_Замыкание IA: каждый FR-нужду закрывает экран, каждый экран ведёт сценарий (см. Key Flows)._

## Voice and Tone
- **Служебно, коротко, по-доброму.** Сообщаем факт, не продаём. Без восклицаний-маркетинга.
- Микрокопирайт (sr, латиница):
  - Уведомление «в пути»: `Vaša porudžbina iz {prodavnica} je u dostavi. Stiže okvirno do {vreme}. Pratite: {link}`
  - Запрос оценки: `Kako je prošla dostava iz {prodavnica}? Ocenite: {link}`
  - Кнопки кабинета: `Dostava je počela` · `Nova dostava` · `Pošalji ponovo`
  - Статусы: `U dostavi` · `Pročitano` · `Isporučeno` · `Nije dostavljeno` · `Spremno`
  - Публичная: `Zdravo, {ime} 👋` · `Vaša porudžbina je u dostavi` · `Stiže okvirno do` · `Kako je prošla dostava?` · `Hvala!`
  - Opt-out (в SMS): короткая инструкция отписки.

## Component Patterns (поведение)
- **delivery_card** (визуал — `{components.delivery_card}`): тап по карточке → деталь. Карточка в «Spremno» несёт primary-кнопку «Dostava je počela» прямо в списке (действие без захода в деталь — экономит шаг «на бегу»).
- **start action** — тап «Dostava je počela» → оптимистичный отклик: кнопка → спиннер, по успеху карточка переезжает в «U dostavi» + тост `Kupac obavešten · stiže do {vreme}`. Идемпотентно (повторный тап не шлёт второй раз).
- **status_chip** — отражает фактический статус уведомления (из Notification). `Pročitano` появляется только если канал подтвердил read (Viber).
- **rating_stars** — на публичной странице после доставки; тап по звезде = отправка (без отдельной кнопки), затем «Hvala!». Повторная оценка не плодит дубли.
- **status_stepper** — 3 шага; текущий подсвечен `{colors.brand}`.

## State Patterns
- **Пусто (нет доставок сегодня):** дружелюбная пустышка + крупная «＋ Nova dostava».
- **Загрузка (расчёт ETA при старте):** спиннер на кнопке (цель < 5 c, NFR). Не блокируем весь экран.
- **Ошибка геокода (адрес не распознан):** подсказка уточнить адрес / поставить точку на карте; fallback — ручной ввод ETA (поток не рвётся).
- **Недоставлено (Nije dostavljeno):** карточка/чип красные; действие «исправить номер + Pošalji ponovo».
- **Получатель отписан:** отправка заблокирована, в кабинете отметка «primalac otkazao obaveštenja».
- **Оценка отправлена:** публичная страница → «Hvala!»; в кабинете у доставки — звёзды.
- **Вне окна 08:00–22:00:** запрос оценки откладывается (пользователю незаметно).

## Interaction Primitives
- **Тач-цели ≥ 48px** (улица/перчатки); главные действия — на всю ширину, прижаты к низу.
- **Одноручный режим:** ключевые действия в нижней трети экрана.
- **Подтверждение необратимого:** «Dostava je počela» шлёт реальное сообщение → лёгкое подтверждение/undo-тост (несколько секунд) перед фиксацией, чтобы случайный тап не ушёл клиенту.
- **Частичные обновления** (HTMX/Alpine): статусы/оценки обновляются без перезагрузки.

## Accessibility Floor
- Контраст текста на светлом ≥ WCAG AA (`{colors.ink}` на `{colors.surface}`/`{colors.bg}`).
- **Статус не только цветом:** всегда цвет + текст/иконка (дальтоники).
- Видимый фокус, навигация с клавиатуры на публичной странице; звёзды-оценка — с aria-label («Ocena {n} od 5»).
- Размер шрифта тела ≥ 15px; уважать системный масштаб.
- Публичная страница работает без JS на чтение статуса (прогрессивное улучшение).

## Key Flows
1. **Milan šalje dostavu (UJ-1).** Главный экран, у «Ana · Spremno» жмёт **Dostava je počela** → спиннер → тост `Kupac obavešten · stiže do 16:00`, карточка уходит в **U dostavi**. **Кульминация:** через минуту видит чип **✓ Pročitano** — клиент увидел.
2. **Ana prati i ocenjuje (UJ-2).** Получает Viber `…u dostavi, stiže do 16:00, {link}` → открывает → видит статус и ETA, готовится принять (горячая пицца, быстрая передача). Через ETA+30 приходит `Kako je prošla dostava?` → открывает → тап **★★★★★**. **Кульминация:** «Hvala!», магазин видит 5★.
3. **Milan ispravlja grešku.** Карточка показывает **Nije dostavljeno** → тап → правит номер → **Pošalji ponovo** → уходит повторно. **Кульминация:** чип меняется на доставлено.
